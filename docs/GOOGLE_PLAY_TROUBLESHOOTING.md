# Google Play Console API 액세스 문제 해결

## ❌ "API 액세스" 페이지에 접근할 수 없어요

### 원인 1: 권한 부족

**증상**:
- URL로 접근해도 페이지가 로드되지 않음
- 403 Forbidden 또는 접근 거부 메시지
- 설정 페이지에 "API 액세스" 탭이 보이지 않음

**해결 방법**:

1. **현재 계정 권한 확인**:
   - Google Play Console → **사용자 및 권한**
   - 자신의 이메일 찾기
   - 권한 레벨 확인

2. **권한 요청**:
   - 계정 소유자에게 연락
   - "API 액세스 페이지 접근 권한" 또는 "소유자/관리자 권한" 요청

3. **권한이 없다면**:
   - 계정 소유자가 대신 서비스 계정 생성
   - JSON 키 파일을 안전하게 전달받기

### 원인 2: 앱이 등록되지 않음

**증상**:
- Google Play Console에 앱이 없음
- API 액세스 메뉴가 비활성화됨

**해결 방법**:
1. 먼저 앱을 등록하세요
2. 앱 등록 후 API 액세스 페이지 접근 시도

### 원인 3: 결제 프로필 미설정

**증상**:
- 일부 기능이 제한됨
- API 액세스가 비활성화됨

**해결 방법**:
1. **설정** → **수익 창출** → **결제 프로필** 설정
2. 결제 프로필 완료 후 API 액세스 접근 시도

## 🔍 권한 확인 방법

### 1단계: 사용자 및 권한 페이지 확인

1. Google Play Console 로그인
2. 왼쪽 사이드바 → **사용자 및 권한** 클릭
3. 자신의 이메일 찾기
4. 권한 레벨 확인:
   - ✅ **소유자 (Owner)**: 모든 권한
   - ✅ **관리자 (Admin)**: 대부분의 권한 (API 액세스 포함)
   - ⚠️ **개발자 (Developer)**: 제한된 권한 (API 액세스 불가능할 수 있음)

### 2단계: 권한 요청

권한이 부족하다면:

1. 계정 소유자에게 이메일 또는 메시지 전송
2. 요청 내용:
   ```
   안녕하세요,
   
   Google Play Console에서 API 액세스 페이지에 접근하려고 하는데
   권한이 부족한 것 같습니다. 
   
   CI/CD 자동 배포를 위해 서비스 계정을 생성해야 합니다.
   소유자 또는 관리자 권한을 부여해주실 수 있을까요?
   
   감사합니다.
   ```

## 🛠️ 대안: 계정 소유자가 대신 생성

권한이 없다면, 계정 소유자가 다음을 수행할 수 있습니다:

### 소유자가 해야 할 일

1. **API 액세스 페이지 접근**
2. **서비스 계정 생성** (상세 가이드 참고)
3. **JSON 키 다운로드**
4. **JSON 파일을 안전하게 전달**
   - 암호화된 메시지로 전송
   - 또는 안전한 파일 공유 서비스 사용

### 받은 후 해야 할 일

1. JSON 파일을 안전하게 보관
2. GitHub Secrets에 추가:
   - Secret 이름: `GOOGLE_PLAY_SERVICE_ACCOUNT_JSON`
   - 값: JSON 파일 전체 내용

## 📞 추가 도움

여전히 문제가 해결되지 않으면:

1. **Google Play Console 도움말**:
   - Google Play Console → **도움말** 메뉴
   - "API 액세스" 또는 "서비스 계정" 검색

2. **Google 지원팀 문의**:
   - Google Play Console 지원팀에 문의
   - 권한 문제나 기술적 문제 해결 요청

3. **공식 문서 확인**:
   - [Google Play Developer API 가이드](https://developers.google.com/android-publisher/getting_started)
   - [서비스 계정 생성 가이드](https://developers.google.com/android-publisher/getting_started#using_a_service_account)
